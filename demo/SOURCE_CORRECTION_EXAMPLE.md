# Source Correction Example

This public-safe example shows how a finished demo fork can invite corrections without turning a GitHub issue into medical advice, legal advice, emergency response training, field operations, outreach, or private-event logistics.

> Safety note: this example is educational and fictional. It does not diagnose, prescribe, certify, dispatch volunteers, identify a real event, or replace emergency services. External action remains **closed until human approval**.

## Example issue body

**Correction lane:** source-backed wording correction

**What should change?**

Replace a vague sentence such as:

> "Offer overdose prevention support at events."

with a sourced, lower-claim wording such as:

> "Share public overdose-response education resources from local public health agencies and national harm-reduction organizations; call emergency services in an emergency."

**Why?**

The revised wording avoids implying certification, medical authority, guaranteed safety, or live field operations. It keeps the repo in documentation-only mode until qualified local humans approve any real-world program.

**Public sources to cite before merging:**

- SAMHSA National Helpline: https://www.samhsa.gov/find-help/national-helpline
- National Harm Reduction Coalition: https://harmreduction.org/issues/overdose-prevention/
- DanceSafe education resources: https://dancesafe.org/education/
- Local public health agency URL: `[ADD OFFICIAL PUBLIC URL]`

**Private information not allowed in the issue:**

- real event addresses, venue backdoors, security plans, or private logistics
- attendee, volunteer, patient, or incident details
- credentials, tokens, payment links, sponsorship asks, or private media
- claims that this repo provides medical/legal advice or training certification

## Human approval gate before external action

Before publishing any external correction, outreach, flyer, training claim, or field-operation plan, an awake operator must answer:

1. What exact text will be public?
2. Which public source backs each health/safety statement?
3. Who reviewed the wording for local accuracy and scope?
4. What private details were removed or redacted?
5. What rollback/correction path exists if a source changes?

If any answer is missing or ambiguous, keep the issue as a documentation-only draft and leave all external lanes `closed_until_human_yes`.
